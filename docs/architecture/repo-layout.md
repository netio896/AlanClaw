# AlanClaw 仓库布局说明

## 设计原则

当前仓库是内容优先的全栈产品仓库。先把专家内容资产、导入契约和未来消费边界固定下来，再逐步接入实际应用。

## 目录职责

### docs/

产品、运营、导入和架构说明文档。

### schemas/

结构化数据的 schema，供后台导入校验和未来共享包使用。

### data/experts/

机器消费优先的数据文件：

- CSV 适合批量导入
- JSON 适合程序消费和校验

### content/experts/

人工编辑优先的专家详情源文件，每个专家一个 Markdown 文件。

### apps/web/

未来公开专家广场应用：

- 专家卡片列表
- 搜索和筛选
- 专家详情页
- 添加/安装专家流程

### apps/admin/

未来内部管理后台：

- 专家录入
- 专家审核
- 批量导入导出
- 分类与标签管理

### packages/expert-catalog/

未来共享数据层：

- 读取 JSON/CSV/Markdown
- schema 校验
- 转换为前台/后台所需视图模型

## 当前边界

- 本次提交不包含可执行应用
- 本次提交不引入运行时依赖
- 本次提交的重点是内容源、数据契约和仓库结构稳定性
