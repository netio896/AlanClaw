# AlanClaw Web and Admin Usage Guide

本手册说明当前本地版本如何使用 AlanClaw Web 专家广场和 Admin 管理台。它只覆盖已经实现的功能，不包含登录、权限、云部署或真实文件上传。

## 启动本地服务

在项目根目录运行：

```powershell
npm run dev
```

打开：

- Admin 管理台: <http://127.0.0.1:4176/>
- Web 专家广场: <http://127.0.0.1:4176/web/index.html>

验证当前版本：

```powershell
npm run smoke
```

## Web 专家广场怎么用

Web 端面向最终用户，用来浏览行业团队、添加专家，并查看自己的专家工作区。

### 首次使用路径

1. 打开 Web 专家广场。
2. 先阅读首页顶部的 3 步说明。
3. 在“首次使用”引导中选择一个行业团队。
4. 点击“添加这套团队”。
5. 页面会提示“已添加到我的专家”。
6. 点击“去查看”，或切换到底部“我的专家”。
7. 在“我的专家”中确认团队成员已经添加。

如果用户关闭了首次引导，可以点击顶部 `?` 或首页的“重新查看引导”再次打开。

### 浏览行业团队

在首页选择“行业团队”视图后，可以查看当前预设团队：

- 建筑行业 Agent Team
- 电商运营 Agent Team
- 社交媒体营销 Agent Team

进入团队详情后，可以看到：

- 团队行业
- 团队摘要
- 典型用例
- 推荐专家成员
- 每位成员在团队中的角色和加入原因

点击“添加整套团队”会把该团队的推荐专家加入“我的专家”。

### 浏览单个专家

在首页选择“推荐专家”“热门精选”或“最近上新”视图后，可以按分类和搜索词筛选专家。

专家卡片支持：

- 点击“添加”加入“我的专家”
- 点击详情箭头查看专家详情
- 在详情里再次添加或移出专家

### 使用“我的专家”

底部切换到“我的专家”后，可以查看已经添加的专家。

如果还没有添加专家，页面会提示先回到专家页添加。已添加专家可以继续打开详情，也可以从卡片上移除。

### 使用“文件”

当前文件页还没有真实文件上传能力。它提供的是文件场景的专家入口：

- 文档总结与问答专家
- Excel 数据整理专家
- 资料整理大师

用户可以在这里直接查看详情或添加这些专家到“我的专家”。

## Admin 管理台怎么用

Admin 端面向维护者，用来编辑专家目录和行业团队模板。保存会写回本地数据文件，并重新生成 Web/Admin 使用的数据文件。

### 专家目录管理

1. 打开 Admin 管理台。
2. 保持在“专家目录”工作区。
3. 从左侧专家列表选择一位专家。
4. 在中间编辑标题、分类、摘要、关于我、系统提示词、标签、语言、渠道、精选状态和排序。
5. 查看右侧“保存前变更”和“导入校验”。
6. 确认无异常后点击“保存到本地”。
7. 回到 Web 刷新，查看专家卡片和详情变化。

专家目录保存后会写回：

```text
data/experts/alanclaw-experts.json
```

并重新生成：

```text
data/experts/alanclaw-experts.csv
apps/web/expert-data.js
apps/admin/expert-data.js
content/experts/*.md
```

### 导入和导出专家

右侧“导入 / 导出”区域支持：

- 导入 JSON 或 CSV
- 导出 JSON
- 导出 CSV

导入会先进入预览，不会立刻写盘。需要确认右侧变更和校验后，再点击“保存到本地”。

### 行业团队模板管理

1. 切换到“行业团队”工作区。
2. 从左侧选择一个团队模板。
3. 修改团队标题、行业、摘要、用例、语言、渠道、精选状态和排序。
4. 在“推荐专家成员”中修改成员标识、角色和加入原因。
5. 可以点击“添加成员”新增推荐专家，也可以点击“删除成员”移除成员。
6. 查看右侧团队校验和保存前变更。
7. 确认无异常后点击“保存团队模板”。
8. 回到 Web 刷新，查看行业团队详情和推荐成员变化。

团队模板保存后会写回：

```text
data/team-templates/alanclaw-team-templates.json
```

并重新生成：

```text
apps/web/team-data.js
apps/admin/team-data.js
```

### 团队模板校验规则

保存团队模板前会检查：

- 团队 `slug` 唯一
- `sort_order` 唯一
- `title`、`industry`、`card_summary` 非空
- `use_cases` 是非空数组
- `recommended_experts` 是非空数组
- 推荐专家成员的 `slug` 必须存在于专家目录
- `languages_supported` 是非空数组
- `channels_supported` 是非空数组

## 推荐工作流

内容维护建议按这个顺序执行：

1. 用 Admin 修改专家或团队模板。
2. 查看右侧 diff 和校验。
3. 保存到本地。
4. 运行 `npm run smoke`。
5. 打开 Web 检查用户路径。
6. 提交并推送代码。

## 当前范围

当前版本是本地开发产品包，已经支持 Web 浏览、前端本地添加、Admin 编辑、校验、保存和生成派生数据。

当前暂不包含：

- 用户登录
- 权限控制
- 数据库
- 云部署
- 真实文件上传
- 聊天执行 Agent
- 专家版本历史
